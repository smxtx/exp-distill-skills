## B. 分析框架

### B.1 五维分析法

用于全面分析工作任务的框架：

| 维度 | 问题 | 分析要点 |
|------|------|----------|
| **目标** | 为什么要做 | 价值、意义、优先级 |
| **方法** | 怎么做 | 路径、工具、流程 |
| **资源** | 需要什么 | 人力、物力、财力、时间 |
| **执行** | 谁来做、什么时候做 | 角色分工、时间节点 |
| **结果** | 期望什么结果 | 指标、验收标准 |

### B.2 PDCA循环

持续改进的工作方法：

```
Plan（计划）→ Do（执行）→ Check（检查）→ Act（改进）
     ↑                                           │
     └───────────────────────────────────────────┘
```

**Plan - 计划阶段**:
- 分析现状，找到问题
- 设定改进目标
- 制定改进方案

**Do - 执行阶段**:
- 按计划执行
- 记录执行过程
- 及时发现问题

**Check - 检查阶段**:
- 对比实际与计划
- 分析差异原因
- 总结经验教训

**Act - 改进阶段**:
- 标准化有效方法
- 推广成功经验
- 识别新问题，进入下一个循环

### B.3 复盘四步法

深度反思的工作方法：

**1. 回顾目标**
- 原始目标是什么
- 目标是否达成
- 是否有新目标

**2. 评估结果**
- 实际结果如何
- 与目标的差距
- 有哪些意外发现

**3. 分析原因**
- 成功因素是什么
- 失败因素是什么
- 根本原因是什么

**4. 总结经验**
- 有哪些可复用经验
- 需要改进什么
- 下次如何做

### B.4 决策矩阵法

用于复杂决策的分析工具：

| 选项 | 权重 | 评分(1-10) | 加权分 |
|------|------|------------|--------|
| 选项A |      |            |        |
| 选项B |      |            |        |
| 选项C |      |            |        |

**权重定义**:
- 成本权重: X%
- 收益权重: Y%
- 风险权重: Z%
- 时间权重: W%

**决策标准**: 加权分最高者为推荐选择

### B.5 黄金圈法则

用于深度思考的框架：

```
Why（为什么）→ How（怎么做）→ What（做什么）
     ↓              ↓              ↓
   目的           方法           结果
```

**Why - 为什么**:
- 为什么做这件事
- 动机是什么
- 意义在哪里

**How - 怎么做**:
- 如何实现目标
- 采用什么方法
- 有什么独特做法

**What - 做什么**:
- 具体做什么
- 产出什么
- 交付什么

## C. 案例库

### C.1 方法论案例：问题拆解法

**问题拆解的"三步法"**：

```
大问题 → 子问题 → 可执行任务
```

**案例应用**：

原始问题："如何提升团队效率"

拆解步骤：

1. **定义问题边界**
   - 是流程问题？人员问题？工具问题？
   - 是协作效率？个人效率？
   - 是某类任务效率低？还是整体效率低？

2. **拆分问题要素**
   - 沟通层：信息传递是否清晰及时
   - 流程层：流程是否冗余低效
   - 技术层：工具是否顺手
   - 人员层：能力是否匹配
   - 管理层：激励是否有效

3. **制定行动项**
   - 沟通层: 建立每日站会制度
   - 流程层: 梳理并简化审批流程
   - 技术层: 引入协作文档工具
   - 人员层: 安排技能培训
   - 管理层: 调整绩效考核方式

**提炼成方法论**：

> **问题拆解三步法**
>
> 适用场景: 面对复杂模糊的问题时
>
> 核心步骤:
> 1. 边界定义：明确问题范围和类型
> 2. 要素拆分：从多维度分解问题
> 3. 行动落地：为每个子问题制定可执行任务
>
> 常见陷阱:
> - 拆分过细：导致行动项过多，无法聚焦
> - 拆分不够：子问题仍过于复杂
>
> 最佳实践: 拆分到每个子问题可以在1-2天内完成

### C.2 经验教训案例：沟通失误

**背景**：
在一次项目评审中，向领导汇报进度时，因为只汇报了问题，没有准备好解决方案，被领导质疑能力。

**问题分析**：

| 分析维度 | 问题 | 分析 |
|----------|------|------|
| 目标 | 为什么汇报 | 寻求支持/同步进度 |
| 方法 | 怎么汇报 | 只说问题，没方案 |
| 结果 | 结果如何 | 被质疑，失去信任 |

**根本原因**：
- 只站在自己角度思考（问题导向）
- 没有考虑领导的期望（方案导向）
- 缺乏汇报的框架意识

**教训总结**：

> **汇报问题三原则**
>
> 1. **永远带方案**：汇报问题时，至少准备一个解决方案
> 2. **提供选项**：如果有多个方案，提供选项并说明利弊
> 3. **明确诉求**：清楚说明需要领导做什么决定或提供什么支持

**实践改进**：

### C.3 成功经验案例：有效会议

**背景**：
团队之前会议效率低下，每次会议都超时但产出不多。后来改进了会议流程，效果显著。

**改进前问题**：
- 会议没有明确目标
- 讨论发散，偏离主题
- 没有主持人控制节奏
- 决议不明确
- 会后没人跟进

**改进措施**：

1. **会前准备**
   - 明确会议目标和议程
   - 提前发送相关材料
   - 确认必要参与者

2. **会中控制**
   - 准时开始
   - 严格按照议程
   - 主持人控制节奏
   - 记录关键决议

3. **会后跟进**
   - 发送会议纪要（24小时内）
   - 明确行动项和负责人
   - 设置下次检查点

**效果衡量**：

| 指标 | 改进前 | 改进后 |
|------|--------|--------|
| 平均时长 | 90分钟 | 45分钟 |
| 议题完成率 | 60% | 95% |
| 行动项跟进率 | 40% | 90% |
| 参会满意度 | 3.2/5 | 4.5/5 |

**提炼成方法论**：

> **高效会议五步法**
>
> 适用场景: 组织或参与任何会议
>
> 步骤:
> 1. 明确目标：会议要解决什么问题
> 2. 准备议程：提前规划讨论内容
> 3. 控制节奏：严格按照时间表
> 4. 记录决议：明确每个决议
> 5. 跟进落实：会后跟踪执行
>
> 核心理念: 无目标不开会，无结论不结束，无跟进不开会

## D. 练习指南

### D.1 每日练习

**练习1：每日三问**

每天结束时问自己三个问题：

1. 今天最重要的收获是什么？
2. 今天最大的挑战是什么？如何应对的？
3. 明天最重要的行动是什么？

**练习2：五分钟记录**

每天花5分钟快速记录：
- 完成的一件事
- 遇到的一个问题
- 学到的一点东西

### D.2 周度练习

**练习3：周度回顾**

每周五花30分钟进行周度回顾：

1. 回顾本周的目标完成情况
2. 分析达成/未达成的原因
3. 提炼一个可复用的经验
4. 规划下周的重点

**练习4：经验提炼**

选择一个本周的成功案例，尝试提炼出可复用的方法论：

- 这个成功可以复制吗？
- 需要什么条件？
- 适用边界是什么？

### D.3 月度练习

**练习5：技能盘点**

每月末盘点：

- 哪些技能提升了？
- 哪些技能退化了？
- 下月重点提升什么？

**练习6：模式识别**

回顾过去一个月的工作，识别：

- 重复出现的成功模式
- 重复出现的问题模式
- 沟通协作中的特点

## E. 常见问题解答

**Q1: 工作太忙，没时间记录怎么办？**

A1: 记录不在于长，在于坚持。可以从"三句话记录法"开始：
- 做了什么事
- 结果怎么样
- 下次怎么做

**Q2: 记录了很多但不知道怎么提炼怎么办？**

A2: 可以用"主题聚类法"：
- 把同类记录放在一起
- 找出共同点
- 抽象成规律

**Q3: 有些经验感觉矛盾怎么办？**

A3: 标注适用边界：
- 经验A适用于：[场景]
- 经验B适用于：[场景]
- 本质上不矛盾，只是场景不同

**Q4: 如何让记录保持更新？**

A4: 建立触发机制：
- 固定时间触发（每天/每周）
- 事件触发（项目结束/重要会议后）
- 情绪触发（特别有成就感/特别沮丧时）

**Q5: 提炼的方法论太抽象怎么办？**

A5: 用"具体化检查"：
- 能举出具体例子吗？
- 能给出操作步骤吗？
- 能说明适用边界吗？
- 不能的话，继续具体化
